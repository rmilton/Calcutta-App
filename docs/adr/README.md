# ADR Directory

Architecture Decision Records (ADRs) capture decisions that affect structure, reliability, boundaries, or operations.

## Naming

- `NNNN-short-title.md` (example: `0001-doc-control-plane.md`)

## When Required

Create/update an ADR for:

- deployment/runtime model changes
- schema semantic changes
- service boundary changes
- reliability strategy changes
- shared API contract changes

## ADR Format

1. Status
2. Context
3. Decision
4. Consequences
5. Rollback/alternatives
6. References
